=== 2. Premier run CI (run #28412781846)
Statut : ÉCHEC (37m46s) — 3 problèmes identifiés :

. **api-key-pool-gradle** — `gradle-wrapper.jar` gitignoré (non tracké dans git).
  Le clone `--depth 1` ne peut pas le récupérer → `Error: Unable to access jarfile`.
  *Cause* : `.gitignore` ligne 23 : `**/gradle/wrapper/gradle-wrapper.jar`.
  *Fix* : committer le wrapper jar OU générer en CI via `gradle wrapper`.

. **readme-gradle** — `Task 'check' not found in root project 'readme-gradle'`.
  Le projet racine est un consommateur (`plugins { alias(libs.plugins.readme) }`)
  qui n'applique pas `java-library` ni `base` → pas de tâche `check`.
  Les vrais tests sont dans `readme-plugin/` (sous-répertoire avec son propre
  `gradlew` + `build.gradle.kts` + tests Cucumber).
  *Fix* : `runAllTests` doit détecter le sous-projet et lancer `check` dedans.

. **slider-gradle** — même pattern que readme-gradle.
  Consommateur `alias(libs.plugins.slider)` + `alias(libs.plugins.readme)`.
  Vrais tests dans `slider-plugin/`.
  *Fix* : idem readme-gradle.

=== 3. Correction log noise pgvector (session Queens 171)
8 `log.warn` → `log.debug` dans codebase-gradle pour silencer le bruit
pgvector en mode dégradé (tests résilients sans pgvector).
