== EPIC MEM-3 : i18n-contracts — Contrats d'Internationalisation N0

*Découverte* : Session 111 bakery — L'i18n est une préoccupation transverse (bakery, slider, codex, training, magic-stick, document, readme). Mérite un artefact N0 partagé dans workspace-bom.

*Objectif* : Définir les contrats (interfaces + data classes) pour l'internationalisation — codes langue, catalogues, bundles de messages, résolution de locale. Zéro implémentation — chaque borough implémente selon son contexte (Thymeleaf pour bakery, Asciidoctor pour codex, Reveal.js pour slider, etc.).

*Charge* : ~8 pts (~2 sessions)
*Priorité* : P1 — Prérequis pour BKY-I18N et tout autre borough multilingue.
*Pattern* : Tier 0 (zéro dépendance externe, kotlin-stdlib only) — comme `codebase-contracts`, `llm-pool-contracts`, `pipeline-contracts`.

=== Langues supportées (10)

[cols="1,2,3,2,2"]
|===
| Rang | Code ISO 639-1 | Langue | Nom natif | RTL
| 1 | `en` | Anglais (English) | English | Non
| 2 | `zh` | Chinois Mandarin | 中文 | Non
| 3 | `hi` | Hindi | हिन्दी | Non
| 4 | `es` | Espagnol | Español | Non
| 5 | `fr` | Français | Français | Non
| 6 | `ar` | Arabe Standard | العربية | Oui
| 7 | `bn` | Bengali | বাংলা | Non
| 8 | `pt` | Portugais | Português | Non
| 9 | `ru` | Russe | Русский | Non
| 10 | `ur` | Ourdou | اردو | Oui
|===

=== Classes/Interfaces proposées

[cols="1,3,2,2"]
|===
| Classe | Description | Type | Pts

| `SupportedLanguage`
| Data class : `code` (ISO 639-1), `name` (English), `nativeName` (endonyme),
  `rtl` (Boolean), `localeTag` (ex: `"fr-FR"`, `"ar-SA"`).
| data class
| 1

| `LanguageCatalog`
| Object : liste statique des 10 `SupportedLanguage`, `all()` → List,
  `findByCode(code)` → SupportedLanguage?, `supportedCodes()` → Set<String>.
| object
| 1

| `I18nConfig`
| Data class : `activeLanguage` (String, default `"fr"`),
  `supportedLanguages` (List<String>, default `["fr"]`),
  `fallbackLanguage` (String, default `"en"`).
  `validate()` → Either<Error, Unit> (vérifie que active ∈ supported).
| data class
| 1

| `MessageBundle`
| Interface : `get(key: String, locale: Locale): String?`,
  `keys(): Set<String>`, `locale(): Locale`.
  Contrat pur — chaque borough fournit son implémentation
  (Properties pour Thymeleaf, YAML pour Asciidoctor, JSON pour Reveal.js).
| interface
| 1

| `LocaleResolver`
| Interface : `resolve(config: I18nConfig): Locale`.
  Contrat de résolution en cascade (CLI > properties > DSL > YAML > default).
  `resolveWithFallback(config): Locale` — si active ∉ supported → fallback.
| interface
| 1
|===

| Slider | Slides Reveal.js | `i18n/{code}.json` + JS bundle
| Codex | PDF/EPUB Asciidoctor | Attributs localisés AsciiDoc
| Training | Formulaires Google Forms | Labels multilingues

=== Note architecture

Cet artefact est *strictement* un contrat (interfaces + data classes).
Aucune implémentation — pas de `ResourceBundle`, pas de `Properties.load`,
pas de parsing YAML/JSON. Chaque borough fournit sa propre implémentation
adaptée à son format de template (Thymeleaf, Asciidoctor, Reveal.js, etc.).

Le contrat garantit l'interopérabilité : un `I18nConfig` produit par bakery
peut être consommé par runner-gradle pour orchestrer un site multilingue.

== EPIC MEM-4 : opencode-session-contracts — Protocole Thin Client opencode↔Gradle

*Découverte* : Session 103 codebase brainstorming (2026-06-14) — Architecture thin-client
opencode↔Gradle. opencode devient une UI terminal uniquement, toute la logique métier
(LLM, RAG, tools) est dans des tâches Gradle.

*Objectif* : Définir les contrats N0 pour le protocole de session entre un thin client
(opencode) et le moteur Gradle (codebase VibecodingTask). 7ème artefact N0 dans workspace-bom.

*Charge* : ~8 pts (~2 sessions)
*Priorité* : P1 — Dépend de EPIC V (VibecodingTask moteur) et EPIC X (verify→adapt) dans codebase.
*Pattern* : Tier 0 (zéro dépendance externe, kotlin-stdlib only).

=== Architecture

```
opencode (thin client UI)
    │ SessionPrompt (contrat N0)
    ▼
runner-gradle (N3) — pont thin-client ↔ Gradle engine
    │ VibecodingTask
    ▼
codebase-gradle (N1) — moteur LLM + RAG + tools
```
